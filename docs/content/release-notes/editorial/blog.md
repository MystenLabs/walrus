# The Walrus Blog: A Protocol Growing Up in Public

_An editorial summary of the Walrus blog milestone announcements (2024 to 2025)._

Where the platform changelog tracks the engineering, the blog announcements track the milestones, the moments Walrus chose to mark publicly. Read together, they form a compact origin story: a research-grade idea introduced as a developer preview, validated by a whitepaper and a community, opened to independent operators on Testnet, and finally hardened into a production Mainnet. Each post is a stage in a decentralized network earning the right to be trusted with real data.

## The developer preview and its first iteration

It begins with the introduction of Walrus as a decentralized storage and data-availability protocol, released as a developer preview for Sui builders, explicitly to gather feedback, with the promise of a broader web3 rollout to come. A follow-up Devnet update reports a redeployment that folds in improvements to storage nodes and clients, and is candid about the cost of being early: all stored blobs were wiped, with a clear-eyed note that this could happen again on Devnet and Testnet but pointedly not on the eventual Mainnet. That honesty about impermanence is a recurring, trust-building motif.

## Substance and community

The whitepaper announcement is the intellectual anchor of the set. By then the developer preview was already storing more than 12 TiB of data, and a community event had gathered over 200 developers building on decentralized storage. This is the moment Walrus pairs a working preview with a rigorous design document, the signal that the project intends to be a protocol, not a demo.

## Opening the network: Testnet, twice

The first public Testnet is framed as a genuine milestone: not just running the software, but running it across a set of independent storage nodes that rotate over time through delegated proof of stake, the first real validation of Walrus as a decentralized system rather than a single-operator service. It also brings user-facing functionality around governance, epochs, and blob deletion. A later Testnet v2 redeployment follows, this time openly flagging backwards-incompatible changes and pointing operators to refreshed binaries and configuration, the network learning to evolve without pretending nothing breaks.

## The destination: Mainnet

The arc resolves with the Mainnet announcement. The production network goes live operated by a decentralized set of over 100 storage nodes, with Epoch 1 beginning in late March 2025. Blobs can be published and retrieved, Walrus Sites can be uploaded and browsed, and the live WAL token can be staked and unstaked to determine future committees. The framing is deliberate: on Mainnet the protocol's security properties hold, and Walrus is finally ready for real applications.

## The arc, in one line

The blog reads as a single sentence stretched across a year: Walrus introduced itself as a preview, proved itself with a whitepaper and a community, decentralized itself across independent operators on Testnet, and graduated to a production Mainnet built to be relied upon.
