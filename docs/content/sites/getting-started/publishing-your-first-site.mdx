The `site-builder` uploads your web framework's output directory to Walrus and stores the metadata on Sui. Your directory needs an `index.html` file at the root to serve as the entry point. You can find various examples in the [Walrus Sites example repository](https://github.com/MystenLabs/example-walrus-sites).

To learn the basics of publishing and updating a Walrus Site, deploy the `walrus-snake` game from the example repository.

## Clone and deploy `walrus-snake`

<Tabs>
<TabItem value="prereq" label="Prerequisites">

- [x] [Install and configure Walrus](/docs/getting-started/index).

- [x] [Install and configure the `site-builder`](/docs/sites/getting-started/installing-the-site-builder).

</TabItem>
</Tabs>

First, clone the Walrus Sites example repository:

```sh
$ git clone https://github.com/MystenLabs/example-walrus-sites.git && cd example-walrus-sites
```

Since the `walrus` and `site-builder` binaries and configuration files are placed in their default locations, you can immediately use the [`deploy` command](/docs/sites/getting-started/using-the-site-builder#deploy) to publish the `./walrus-snake` site:

```sh
$ site-builder --context=testnet deploy ./walrus-snake --epochs 1
```

Depending on the network, the duration of an epoch might vary. Visit the official [Walrus network release schedule](https://www.walrus.xyz/network-release-schedule) to view the most up-to-date epoch duration details.

## View your deployed site

How you access your site depends on which network you used:

- **Mainnet sites**: Can be accessed through any Mainnet portal. https://wal.app serves Walrus Sites on Mainnet by resolving SuiNS names that point to them.

- **Testnet sites**: Can be accessed through any Testnet portal. Walrus Foundation does not operate a Testnet portal. You can [self-host or run one locally](/docs/sites/portals/deploy-locally).

This output tells you that, for each file in the folder, a new [Walrus quilt](/docs/system-overview/quilt) was created with a respective quilt ID. The configuration file is necessary to ensure that the `site-builder` knows the correct Sui package for the Walrus Sites logic.

More details on the configuration of the `site-builder` can be found in the [advanced configuration](/docs/sites/getting-started/configuration-file-reference) section.

## Update the site

To update your site content, like changing the title from "eat all the blobs!" to "Glob all the Blobs!", make the edit in the `./walrus-snake/index.html` file.

You can then update the existing site by running the `deploy` command again. The `deploy` command uses the site object ID stored in `ws-resources.json` from the initial deployment to identify which site to update. You do not need to specify the object ID manually:

```sh
site-builder --context=testnet deploy --epochs 1 ./walrus-snake
```

The wallet you are using must be the owner of the Walrus Site object to be able to update it.

Browsing the provided URL should reflect any changes you have made to the site.

During updates, the `site-builder` removes and re-uploads all site resources together as a single quilt, even files you haven't modified. This approach offers significant benefits, including faster upload speeds and lower storage costs, especially when uploading many small files. The drawback is that you cannot update files individually; changing one file means re-uploading the entire quilt.
