The Walrus Sites `site-builder` uploads your web framework's output directory to [Walrus](https://www.walrus.xyz/), where your site resources (HTML, CSS, JavaScript, and images) are stored as blobs. A [Sui smart contract](https://docs.sui.io/guides/developer/packages/move-package-management) acts as an on-chain index, mapping each resource path to its corresponding Walrus blob ID and recording site ownership. [Portals](/docs/sites/portals/deploy-locally) use this on-chain metadata to resolve and serve your site to visitors. 

Your directory needs an `index.html` file at the root to serve as the entry point. You can find various examples in the [Walrus Sites example repository](https://github.com/MystenLabs/example-walrus-sites).

## Clone and deploy `walrus-snake`

To learn the basics of publishing and updating a Walrus Site, deploy the `walrus-snake` game from the Walrus Sites example repository.

First, clone the example repository:

```sh
$ git clone https://github.com/MystenLabs/example-walrus-sites.git && cd example-walrus-sites
```

Then, [`deploy`](/docs/sites/getting-started/using-the-site-builder) the `walrus-snake` game:

```sh
$ site-builder --context=testnet deploy ./walrus-snake --epochs 1
```

Depending on the network, the duration of an epoch might vary. Currently on Walrus Testnet, the duration of an epoch is 1 day. On Mainnet, the duration of an epoch is 2 weeks.

## View your deployed site

How you access your site depends on which network you used:

- **Mainnet sites**: Can be accessed through any Mainnet portal. To access your Walrus Site through https://wal.app, a [SuiNS](https://suins.io/) name is required.

- **Testnet sites**: Can be accessed through a [self-hosted or local portal](/docs/sites/portals/deploy-locally). Walrus Foundation does not operate a Testnet portal.

## Update the site

During updates, the `site-builder` removes and re-uploads all site resources together as a single quilt, even files you haven't modified. This approach offers significant benefits, including faster upload speeds and lower storage costs, especially when uploading many small files. The drawback is that you cannot update files individually; changing one file means re-uploading the entire quilt.

:::info

If you are building a Walrus Site using a web framework, ensure you rebuild your website before updating the corresponding Walrus Site.

:::

To update the example `walrus-snake` site, like changing the title from "eat all the blobs!" to "Glob all the Blobs!", make the edit in the `./walrus-snake/index.html` file.

You can then update the existing site by running the `deploy` command again:

```sh
site-builder --context=testnet deploy --epochs 1 ./walrus-snake
```

The `deploy` command uses the site object ID stored in `ws-resources.json` from the initial deployment to identify which site to update. If you want to update a different site than the one in `ws-resources`, you can specify a different object ID using the `--object-id <OBJECT_ID>` flag.

Additionally, the wallet you are using must be the owner of the Walrus Site object to be able to update it.
