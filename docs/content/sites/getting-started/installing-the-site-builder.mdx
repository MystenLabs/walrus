The `site-builder` is a Rust CLI tool that enables you to create, edit, and publish a Walrus Site. You can install the `site-builder` using one of the pre-built binaries, or by using [`suiup`](https://github.com/MystenLabs/suiup).

<Tabs>
<TabItem value="prereq" label="Prerequisites">

- [x] Install a recent version of [Rust](https://www.rust-lang.org/tools/install).

- [x] [Install and configure Walrus](/docs/getting-started/index).

</TabItem>
</Tabs>

## Installation

Similar to the Walrus client CLI tool, the `site-builder` client binary is currently provided for Ubuntu, macOS (Apple and Intel CPUs), and Windows.

:::info

While a pre-built binary is offered for Windows, the installation instructions assume a UNIX-based system for directory structure, commands, and so on. If you are using Windows, you might need to adapt most of those.

:::

You can also use a `curl` request to download the latest build from the Google Cloud Storage (GCS) bucket. Ensure you set the `$SYSTEM` variable to your current system: `ubuntu-x86_64`, `ubuntu-x86_64-generic`, `macos-arm64`, `macos-x86_64`, `windows-x86_64.exe`.

<Tabs>
<TabItem value="mainnetcurl" label="Mainnet curl request">

```sh

$ chmod +x site-builder
```

</TabItem>

To be able to run it as simply `site-builder`, move the binary to any directory included in your `$PATH` environment variable. Standard locations are `/usr/local/bin/`, `$HOME/bin/`, or `$HOME/.local/bin/`.

After this is done, you should be able to simply type `site-builder` in your terminal.

```sh
$ site-builder
```

If successful, the console responds:

```sh
Usage: site-builder [OPTIONS] <COMMAND>

  deploy   Deploy a new site on Sui
  publish  Publish a new site on Sui

  help     Print this message or the help of the given subcommand(s)
```

### Using `suiup`

Alternatively, you can also install the `site-builder` by using the `suiup` tool, which enables you seamlessly install different versions.

First, install `suiup`:

```sh
$ curl -sSfL https://raw.githubusercontent.com/Mystenlabs/suiup/main/install.sh | sh
```

Then, install the `site-builder`:

```sh
$ suiup install site-builder@mainnet
```

## Configuration

The `site-builder` tool requires the [configuration file](/docs/sites/getting-started/configuration-file-reference) `sites-config.yaml` to work properly.

You can use `sites-config.yaml` to define different contexts and their configurations. The only required field is `package ID` as it represents the Sui object ID of the Walrus Sites smart contract. You can find the latest package ID in the [Walrus Sites repository](https://github.com/MystenLabs/walrus-sites/tree/mainnet) on the `mainnet` branch.

By default, the `site-builder` looks for the configuration file in the current directory, `$XDG_CONFIG_HOME/walrus/`, `~/.config/walrus/`, or `~/.walrus/`.

Download the `sites-config.yaml` file from the repository, and place it in one of the aforementioned default locations. For example:

```sh
$ curl https://raw.githubusercontent.com/MystenLabs/walrus-sites/refs/heads/mainnet/sites-config.yaml -o ~/.config/walrus/sites-config.yaml
```

If you wish to specify a different `sites-config.yaml` location, use the `--config` flag to specify the path to the configuration file.

```sh
$ site-builder --config /path/to/sites-config.yaml publish <build-directory-of-a-site>
```
